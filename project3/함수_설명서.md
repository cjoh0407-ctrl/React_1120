## 📂 App.js - 앱의 중심, 데이터 관리자

### 1. `reducer(state, action)`

**무엇을 하나요?**
일기 데이터를 추가하거나 수정하거나 삭제할 때, 어떻게 처리할지 결정하는 함수입니다.

**쉽게 설명하면:**
학급 출석부를 관리하는 선생님이라고 생각해봅시다. 
- 새 학생이 전학 오면 이름을 추가하고 (CREATE)
- 학생 정보가 바뀌면 수정하고 (UPDATE)
- 학생이 전학 가면 이름을 지웁니다 (DELETE)

**어떻게 동작하나요?**
```
1단계: action을 받습니다 (예: "일기를 추가해줘")
2단계: action.type을 확인합니다 (CREATE인지, UPDATE인지, DELETE인지)
3단계: type에 따라 다르게 처리합니다
   - CREATE: 새 일기를 맨 앞에 추가
   - UPDATE: 일치하는 id의 일기를 새 내용으로 교체
   - DELETE: 일치하는 id의 일기를 제거
   - INIT: 처음 시작할 때 데이터 설정
4단계: 새로운 일기 목록을 반환합니다
```

**예시:**
```javascript
// 일기 3개가 있는 상태
state = [일기1, 일기2, 일기3]

// CREATE 액션이 오면
action = { type: "CREATE", data: 새일기 }
// 결과: [새일기, 일기1, 일기2, 일기3] - 맨 앞에 추가!

// DELETE 액션이 오면
action = { type: "DELETE", targetId: "mock2" }
// 결과: [일기1, 일기3] - 일기2가 사라짐!
```

### 2. `onCreate({date, content, emotionId})`

**무엇을 하나요?**
새로운 일기를 만드는 함수입니다.

**쉽게 설명하면:**
공책에 새 일기를 쓰는 것과 같습니다. 날짜, 오늘의 기분, 내용을 적으면 공책에 기록되는 거죠.

**어떻게 동작하나요?**
```
1단계: 날짜, 내용, 감정을 받습니다
2단계: 새 일기를 만듭니다
   - id: 현재 번호를 사용 (예: 0, 1, 2, ...)
   - date: 날짜를 숫자로 변환 (컴퓨터가 이해하기 쉽게)
   - content: 일기 내용 그대로
   - emotionId: 감정 번호 (1=완전좋음, 5=끔찍함)
3단계: reducer에게 "CREATE"로 일기를 추가해달라고 요청
4단계: 다음 일기를 위해 번호를 1 증가 (0 → 1 → 2 → ...)
```

**예시:**
```javascript
// 사용자가 입력한 내용
onCreate({
  date: "2024-12-09",
  content: "오늘 시험 잘 봤다!",
  emotionId: 1
})

// 만들어지는 일기
{
  id: 0,                    // 첫 번째 일기라서 0
  date: 1702123456789,      // 날짜를 숫자로 변환
  content: "오늘 시험 잘 봤다!",
  emotionId: 1              // 완전 좋음
}
```

### 3. `OnUpdate({id, date, content, emotionId})`

**무엇을 하나요?**
이미 쓴 일기를 수정하는 함수입니다.

**쉽게 설명하면:**
공책에 쓴 일기를 지우개로 지우고 다시 쓰는 것과 같습니다. 날짜나 내용을 바꾸고 싶을 때 사용합니다.

**어떻게 동작하나요?**
```
1단계: 수정할 일기의 id와 새로운 내용을 받습니다
2단계: 새 데이터를 만듭니다
   - id는 그대로 (어떤 일기를 수정할지 알아야 하니까)
   - date, content, emotionId는 새로운 값으로
3단계: reducer에게 "UPDATE"로 일기를 바꿔달라고 요청
```

**예시:**
```javascript
// 기존 일기
{id: "mock1", date: 어제, content: "심심했다", emotionId: 3}

// 수정 요청
OnUpdate({
  id: "mock1",
  date: "2024-12-09",
  content: "친구랑 놀았다!",
  emotionId: 2
})

// 결과: mock1 일기의 내용이 바뀜
{id: "mock1", date: 오늘, content: "친구랑 놀았다!", emotionId: 2}
```

---

### 4. `OnDelete(targetId)`

**무엇을 하나요?**
일기를 삭제하는 함수입니다.

**쉽게 설명하면:**
공책에서 페이지를 찢어내는 것과 같습니다. 해당 일기를 완전히 없애버립니다.

**어떻게 동작하나요?**
```
1단계: 삭제할 일기의 id를 받습니다
2단계: reducer에게 "DELETE"로 일기를 지워달라고 요청
3단계: reducer가 해당 id의 일기를 목록에서 제거합니다
```

**예시:**
```javascript
// 일기 목록
[일기1(id: mock1), 일기2(id: mock2), 일기3(id: mock3)]

// mock2를 삭제
OnDelete("mock2")

// 결과
[일기1(id: mock1), 일기3(id: mock3)]  // 일기2가 사라짐!
```

## 📂 Home.js - 홈 화면

### 1. `onIncreaseMonth()`

**무엇을 하나요?**
다음 달로 이동하는 함수입니다.

**쉽게 설명하면:**
달력을 넘기는 것과 같습니다. 12월에서 ">" 버튼을 누르면 1월로 넘어갑니다.

**어떻게 동작하나요?**
```
1단계: 현재 보고 있는 날짜(pivotDate)를 확인합니다
2단계: 월을 1 증가시킵니다 (12월 → 1월, 1월 → 2월)
3단계: 새 날짜로 업데이트합니다
4단계: 화면이 자동으로 새로고침되면서 해당 달의 일기만 보입니다
```

**예시:**
```javascript
// 현재 2024년 12월을 보고 있음
pivotDate = new Date(2024, 11, 1)  // 11 = 12월 (0부터 시작)

// ">" 버튼 클릭
onIncreaseMonth()

// 결과: 2025년 1월로 이동
pivotDate = new Date(2024, 12, 1)  // 12 = 1월
```

---

### 2. `onDecreaseMonth()`

**무엇을 하나요?**
이전 달로 이동하는 함수입니다.

**쉽게 설명하면:**
달력을 뒤로 넘기는 것과 같습니다. 3월에서 "<" 버튼을 누르면 2월로 돌아갑니다.

**어떻게 동작하나요?**
```
1단계: 현재 날짜를 확인합니다
2단계: 월을 1 감소시킵니다 (3월 → 2월, 1월 → 12월)
3단계: 새 날짜로 업데이트합니다
4단계: 해당 달의 일기만 표시됩니다
```

**예시:**
```javascript
// 현재 2025년 1월
pivotDate = new Date(2025, 0, 1)  // 0 = 1월

// "<" 버튼 클릭
onDecreaseMonth()

// 결과: 2024년 12월로 이동
pivotDate = new Date(2024, 11, 1)  // 11 = 12월
```

### 2. `onSubmit(data)`

**무엇을 하나요?**
사용자가 작성한 일기를 저장하는 함수입니다.

**쉽게 설명하면:**

```javascript
  emotionId: 2

onSubmit(data)
// → onCreate 실행
// → 홈으로 이동
// → 새 일기가 목록에 추가됨!
```

## 📂 Editor.js - 일기 작성/수정 화면

### 1. `handleChangeDate(e)`

**무엇을 하나요?**
사용자가 날짜를 선택할 때마다 실행되는 함수입니다.

**쉽게 설명하면:**
달력에서 날짜를 클릭하면, 그 날짜가 일기에 기록됩니다.

**어떻게 동작하나요?**
```
1단계: 사용자가 날짜 입력창을 변경합니다
2단계: e.target.value에 선택한 날짜가 들어있습니다 (예: "2024-12-09")
3단계: setState를 사용해 state.date를 새 날짜로 바꿉니다
4단계: 화면에 새 날짜가 표시됩니다
```

**예시:**
```javascript
// 처음 날짜: 2024-12-08
state.date = "2024-12-08"

// 사용자가 날짜를 2024-12-09로 변경
handleChangeDate(e)  // e.target.value = "2024-12-09"

// 결과
state.date = "2024-12-09"  // 날짜가 바뀜!
```

### 2. `handleChangeContent(e)`

**무엇을 하나요?**
사용자가 일기 내용을 입력할 때마다 실행되는 함수입니다.

**쉽게 설명하면:**
일기장에 글을 쓸 때마다, 쓴 내용이 저장됩니다.

**어떻게 동작하나요?**
```
1단계: 사용자가 textarea에 글자를 입력합니다
2단계: 글자를 입력할 때마다 이 함수가 실행됩니다
3단계: e.target.value에 현재까지 쓴 모든 내용이 들어있습니다
4단계: setState로 state.content를 업데이트합니다
```

**예시:**
```javascript
// 사용자가 타이핑: "오늘"
handleChangeContent(e)  // e.target.value = "오늘"

handleChangeContent(e)  // e.target.value = "오늘은"
state.content = "오늘은"

// 계속 타이핑: "오늘은 좋았다"
handleChangeContent(e)  // e.target.value = "오늘은 좋았다"

```

```
   - "latest" = 최신순
   - "oldest" = 오래된 순
3단계: setSortType으로 정렬 방식을 저장합니다
4단계: useEffect가 자동으로 실행되어 일기를 재정렬합니다
```

**예시:**
```javascript
// 처음엔 최신순
sortType = "latest"
목록: [오늘 일기, 어제 일기, 그저께 일기]

// 사용자가 "오래된 순" 선택
onChangeSortType(e)  // e.target.value = "oldest"

// 결과
sortType = "oldest"
목록: [그저께 일기, 어제 일기, 오늘 일기]  // 순서가 바뀜!
```

### 3. `compare(a, b)` (useEffect 내부 함수)

**무엇을 하나요?**
두 일기를 비교해서 어떤 게 먼저 와야 할지 결정하는 함수입니다.

**쉽게 설명하면:**
두 학생의 키를 비교해서 줄을 세우는 것과 같습니다. 날짜가 큰 게 앞에 오거나, 작은 게 앞에 올 수 있습니다.

**어떻게 동작하나요?**
```
1단계: 두 일기 a와 b를 받습니다
2단계: sortType을 확인합니다
   - "latest"면: b.date - a.date (큰 날짜가 앞으로)
   - "oldest"면: a.date - b.date (작은 날짜가 앞으로)
3단계: 결과값을 반환합니다
   - 양수: b가 a 앞으로
   - 음수: a가 b 앞으로
   - 0: 순서 유지
```

**예시:**
```javascript
// 최신순 정렬
sortType = "latest"
a.date = 100 (그저께)
b.date = 300 (오늘)

compare(a, b)
= b.date - a.date
= 300 - 100
= 200 (양수)
→ b가 a 앞으로 → [오늘, 그저께]

// 오래된 순 정렬
sortType = "oldest"
compare(a, b)
= a.date - b.date
= 100 - 300
= -200 (음수)
→ a가 b 앞으로 → [그저께, 오늘]
```

### 4. `getMonthRangeByDate(date)`

**무엇을 하나요?**
특정 날짜가 속한 달의 시작과 끝 시간을 알려주는 함수입니다.

**쉽게 설명하면:**
"12월"이라고 하면, "12월 1일 00:00:00부터 12월 31일 23:59:59까지"를 알려주는 함수입니다.

**어떻게 동작하나요?**
```
1단계: 날짜를 받습니다 (예: 2024년 12월 15일)
2단계: 그 달의 시작 시간을 계산합니다
   - new Date(2024, 11, 1) → 12월 1일 00:00:00
   - getTime()으로 숫자로 변환
3단계: 그 달의 끝 시간을 계산합니다
   - new Date(2024, 12, 0, 23, 59, 59)
     (12월의 0일 = 11월의 마지막 날 = 12월 31일)
   - getTime()으로 숫자로 변환
4단계: {beginTimeDate, endTimeStamp} 형태로 반환합니다
```

**예시:**
```javascript
// 2024년 12월 15일을 넣으면
const date = new Date(2024, 11, 15)
getMonthRangeByDate(date)

// 결과
{
  beginTimeDate: 1701388800000,  // 2024-12-01 00:00:00
  endTimeStamp:  1704047999000   // 2024-12-31 23:59:59
}

// 이 범위로 일기를 필터링:
data.filter((it) => 
  beginTimeDate <= it.date && it.date <= endTimeStamp
)
// → 12월에 쓴 일기만 나옴!
```

## 🎓 핵심 개념 정리

### 1. 이벤트 핸들러 (handle...)
**무엇인가요?**
사용자가 버튼을 누르거나 글자를 입력할 때 실행되는 함수입니다.

**예시:**
- `handleChangeDate` - 날짜 변경할 때
- `handleSubmit` - 제출 버튼 누를 때
- `handleOnClick` - 클릭할 때

---

### 2. 네비게이션 함수 (go...)
**무엇인가요?**
페이지를 이동시키는 함수입니다.

**예시:**
- `goBack()` - 뒤로가기
- `goEdit()` - 수정 페이지로
- `goDetail()` - 상세보기 페이지로

---

### 3. 콜백 함수 (on...)
**무엇인가요?**
다른 컴포넌트에서 호출하는 함수입니다. 주로 데이터를 전달받아 처리합니다.

**예시:**
- `onCreate` - 새 일기 만들 때
- `onSubmit` - 폼 제출할 때
- `onClick` - 클릭 이벤트 처리

---

## 💡 함수 이름 규칙

- `handle...` - 사용자 입력 처리 (예: handleChangeDate)
- `on...` - 이벤트가 발생했을 때 (예: onClick, onSubmit)
- `go...` - 페이지 이동 (예: goBack, goEdit)
- `get...` - 값을 가져오기 (예: getFormattedDate)
